SOLID Principles

S (Single Responsibility Principle)

According to the Single Responsibility Principle, each class should only have one job. If a class has many responsibilities, it is said to be coupled. When one responsibility is modified, the other responsibility is modified as well.

Take a look at the Multiplication class to the right.

  • which has only one job to get the values from Calculation class and multiply them.

    • Multiplication Class

    class Multiplication(Calculation):

    def get_result(self):
        """get the multiplication results"""
        result = 1.0
        for value in self.values:
            result = result * value
        return result

    I (Interface Segregation Principle)

    According to the Interface Segregation Principle, the principle states that no client should be forced to depend on methods it does not use.

    Take a look at the example to the right. The example is in java coding language.

  • Class called ALLInOnePrinter has 3 methods defined. Majority of printers will be able to use all methods listed.
  • There are some old printers which does not have the functionality of scan or fax.
  • We don't have such printers to throw us errors when they are not supposed to use those methods but still forced to use because they are part of one class.
  • The way to solve this problem is to create separate interfaces and only assign the ones that use the functionality.
  • The OldPrinter Class only uses the print functionality so assign only that method.

    • Example Class

    class AllInOnePrinter : ISmartDevice
{
    public void Print()
    {
         // Printing code.
    }

    public void Fax()
    {
         // Beep booop biiiiip.
    }

    public void Scan()
    {
         // Scanning code.
    }
}
            
    interface IPrinter{
    void Print();
}

interface IFax{
    void Fax();
}

interface IScanner{
    void Scan();
}
#Old printer class
class OldPrinter : IPrinter
{
    public void Print()
    {
         // Printing code.
    }
